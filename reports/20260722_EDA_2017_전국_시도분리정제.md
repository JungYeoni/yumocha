# 2017년 시행계획 세부사업 전국 시도 분리·정제 결과

## 1. 작업 목적과 현재 상태

2017년도 지방자치단체 저출산·고령사회 시행계획의 세부사업 예산표를 17개 시도 단위로 정제한다. 재정대응지수 산출에는 정제된 세부사업 행의 예산 합계를 사용하고, 원본 소계·총계는 결과를 강제로 맞추는 기준이 아니라 차이 금액과 오차율을 확인하는 QA 기준으로 사용한다.

- 실행 노트북: `notebooks/20260720_EDA_2017_전국_시도분리정제.ipynb`
- 입력: `data/raw/칼럼정렬/세부사업표추출_2017년도 지방자치단체 저출산고령사회 시행계획 (제3차 기본계획)_칼럼정렬.xlsx`
- 원본 행: 11,808건
- 연속 완전 중복 제거 후보: 4,763건
- 재원 정리 후 전체 행: 4,873건
- 최종 세부사업 행: 4,551건
- 대분류 결측: 0건
- 중분류 결측: 0건
- 공통/자체 불일치: 0건
- 테스트: 110개 전체 통과
- 진행 상태: 구조·재원·LLM 정제와 wide/long 저장 완료

## 2. 재원행과 셀 분리 중복 정리

2017년 원본은 하나의 사업이 `계/국비/지방비`로 나뉘어 있으며, PDF 표의 병합 셀이 분리되면서 같은 재원행이 2~7회 연속 반복된다. 사업명·예산만 같은 정상 사업을 삭제하지 않기 위해 다음 조건을 모두 만족하는 행만 추출 중복으로 제거했다.

- `원본행`을 제외한 모든 컬럼이 완전히 같음
- 입력 순서상 바로 붙어 있음
- 중복 묶음의 첫 행은 유지함

이후 `select_total_budget_rows()`를 적용했다.

- `계`가 있으면 `계` 1행만 유지
- `계`가 없고 지방비만 있으면 지방비 금액을 총예산으로 사용
- `계`가 없고 국비와 지방비가 있으면 두 금액을 합산
- 원래 재원 구성은 중간 QA용 `재원구분`에 보존
- `재원구분`은 최종 wide/long 스키마에서 제외
- 최종 `사업분류재정구분`은 대분류를 기준으로 `공통/자체`로 재생성

강원은 세부사업 553행에서 276행으로 줄었고, 사업명·예산·주요내용이 모두 같은 잔여 반복행은 0건이다. 예시로 `신혼부부 맞춤형 임대주택 공급 대폭 강화 (춘천시 거두리 행복주택 건설 공급)`은 원본에서 계 3행, 국비 3행, 지방비 2행으로 반복되었으나 정제 후 `계 7,314`만 남았다.

## 3. 지역별 계층 보정

### 3-1. 경기

원본행 5,111~5,142는 전체 예산을 유형별로 다시 보여주는 요약 블록이다. 대분류 전파에 필요한 5,118~5,120의 `Ⅰ. 공통사업`은 유지하고 나머지 요약행은 `헤더반복`으로 제외했다. `1) 자체사업(도)`, `2) 자체사업(시군)`은 프로젝트에서 필요한 별도 계층이 아니며 세부사업이 아닌 요약행으로 처리했다.

### 3-2. 서울

서울 자체사업 구간의 `1. 저출산 대책`, `2. 고령화 대책`은 `( 자체사업 )` 접미사가 없어 기본 정규식으로 중분류 제목으로 인식되지 않았다. 서울 자체사업 구간에 한해 접미사를 복원해 중분류로 전파했다.

### 3-3. 대전

대전은 `Ⅰ/Ⅱ` 로마자 제목이 대분류이고, `1./2./3.` 주제 제목이 중분류이며, `1)/2)` 제목은 하위 소계이다. 다음과 같이 처리했다.

- `N. 주제 (N개 과제)`: `중분류_소계`
- `N) 제목`: 하위 소계이므로 `헤더반복`
- `총계`: `헤더반복`
- 대·중분류의 `(N개 과제)` 표기 제거
- 중분류에 대분류에 따라 `(공통사업)` 또는 `(자체사업)` 부여

결과는 대분류 2개, 중분류 6개, 계층 결측 0건, 공통/자체 불일치 0건이다.

### 3-4. 광주

광주의 `Ⅱ. 자체사업`을 다른 지역과 같은 `Ⅱ. 지자체 자체사업`으로 통일했다. 행 수·예산·중분류는 변경하지 않았다.

### 3-5. 경남

`Ⅰ. 공통사업` 마지막 구간의 `3. 대응기반 강화(자체사업)`은 다음 `Ⅱ. 지자체 자체사업` 시작 전에 있고, 하위 사업에 국비와 지방비가 모두 기재된 원본 접미사 오기다. 경남·공통사업·해당 중분류를 모두 조건으로 묶어 `3. 대응기반 강화(공통사업)`으로 보정했다.

## 4. 비예산 행

재원 정리 후 `비예산`만 있는 세부사업은 127건이다.

- 2017년 예산 0: 103건
- 2017년 예산 결측: 23건
- 2017년 비영 예산: 1건

유일한 비영 예산은 충북 `찾아가는 출산양육프로그램 운영`이다. 원본은 `비예산`, 2017년 예산 1백만 원, 2016년 예산 결측으로 기재되어 있다. 코드로 0으로 추정해 바꾸지 않고 원본 금액 1을 유지했다. 해당 중분류는 원본 소계값이 결측이므로 QA 오차율은 `판정불가`이다.

- 1백만 포함 leaf 합계: 215,198
- 해당 사업 제외 leaf 합계: 215,197
- 원본 소계: 결측

`판정불가_사유`를 전체 연도 QA에 기록하는 후속 작업은 GitHub 이슈 #31로 분리했다.

### 5-1. 경남 공통사업/고령화 대책: 원본 소계 오류

원본 문서에는 다음과 같이 기재되어 있다.

| 구분 | 2016년 | 2017년 | 증감액 | 비율 |
|---|---:|---:|---:|---:|
| 계 | 635,186 | 664,503 | 29,317 | 4.6 |
| 국비 | 193,086 | 198,078 | 4,992 | 2.6 |
| 지방비 | 68,250 | 66,619 | -1,631 | -2.4 |

원본 소계 행은 `계 = 국비 + 지방비`를 만족하지 않는다. 반면 하위 세부사업의 재원별 합계는 다음과 같다.

| QA 항목 | 2017년 금액 |
|---|---:|
| 세부사업 국비 합계 | 664,503 |
| 세부사업 지방비 합계 | 198,078 |
| 세부사업 전체 합계 | 862,581 |
| QA가 읽은 원본 소계 | 664,503 |
| 차이 | 198,078 |
| 오차율 | 29.81% |

원본의 `계 664,503`은 세부사업 국비 합계와 같고, `국비 198,078`은 세부사업 지방비 합계와 같다. 2016년 금액도 같은 패턴이다. 사용자가 원본 문서 표기가 추출 시트와 같음을 확인했으므로, 이 건은 추출 오류가 아니라 `원본 소계 오류`로 기록한다.

정제 데이터에서는 각 세부사업의 `계` 금액을 유지하고 862,581을 합계로 사용한다. 원본 소계 664,503에 맞추기 위해 정상 세부사업을 제거하지 않는다. 이 근거를 재현하는 전용 대조 셀을 노트북 QA 바로 아래에 추가했다.

## 6. LLM 주요내용 정제

Upstage Solar API로 4,551개 leaf 전체를 처리했다. 최초 6개 병렬 실행에서 429 요청 한도 오류가 발생해 중단한 후, API 실패를 원문 완료로 저장하지 않도록 보완하고 동시 요청을 1개로 낮춰 100건 단위 체크포인트로 완료했다.

- 체크포인트: `data/interim/2017_llm_정제_체크포인트.csv`
- 전체 leaf: 4,551건
- 원문 및 LLM 정제문 존재: 4,526건
- 원문 결측: 25건
- 원문과 달라진 정제문: 273건
- 숫자 불일치: 2건 발견 후 원문 복구
- 의미 보존 이상: 3건 발견 후 원문 복구

의미 보존 이상은 정제문에 사업명과 `원문:` 라벨을 추가한 전남 2건과, 원문을 과도하게 축약한 전남 1건이다. 정제문에만 `원문:`이 추가되었거나 원문 대비 길이비가 0.5 미만 또는 1.5 초과인 행을 자동 검출하는 QA를 노트북에 추가했다.

최종 재검증 결과 숫자 불일치, `needs_llm_rerun` 대상, 추가된 `원문:` 라벨, 극단적 길이비, 원문 결측 환각은 모두 0건이다. 검토 파일은 `reports/2017_LLM_숫자불일치_검토.csv`, `reports/2017_LLM_의미보존_검토.csv`에 보존했다.

## 7. 최종 저장 스키마

최종 wide/long 데이터에는 다음 원칙을 적용한다.

- `사업분류재정구분`: `공통/자체`
