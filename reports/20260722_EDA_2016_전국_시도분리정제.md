# 2016년 시행계획 세부사업 전국 시도 분리·정제 결과

## 1. 작업 목적

2016년도 지방자치단체 저출산·고령사회 시행계획의 세부사업 예산표를 17개 시도 단위로 정제한다. 재정대응지수 산출에는 정제된 `세부사업` 행의 예산 합계를 사용하고, 원자료의 소계·총계는 결과를 강제로 맞추는 기준이 아니라 차이 금액과 오차율을 확인하는 QA 기준으로 사용한다.

## 2. 공통 정제 기준

- 원본 예산 컬럼을 보존한 상태에서 숫자형 파생 컬럼을 생성했다.
- `계/국비/지방비`가 별도 행으로 반복되는 2016년 표 구조는 세부사업별 `계` 행을 우선 선택했다.
- `계` 행이 없을 때만 재원별 예산을 합산하고 재원구분을 `계`로 표시했다.
- 같은 사업명이 반복되어도 입력 순서상 연속되고 재원 토큰이 중복되지 않는 행만 하나의 재원 블록으로 합산했다. `국비-지방비-국비-지방비`처럼 재원 토큰이 다시 시작되면 서로 다른 사업으로 보존했다.
- 값이 완전히 같은 정상 사업도 존재하므로 자동 중복 삭제를 중단했다. 원본 표에서 추출 오류임을 확인한 원본행만 `CONFIRMED_DUPLICATE_ROWS`에 명시해 제거하며, 현재 2016년의 확인된 삭제 대상은 없다.
- 단위 표기, 총계, 소계, 합계 행은 세부사업 합계에서 제외했다.
- 실제 세부사업은 원본 소계와 불일치하더라도 임의로 제거하지 않았다.
- 최종 대분류는 17개 지역 모두 `Ⅰ. 공통사업`, `Ⅱ. 지자체 자체사업`으로 표준화했다.

## 3. 지역별 예외 처리

### 3.1 광주

광주는 `Ⅰ. 공통사업`과 같은 로마숫자 대분류 행이 없고, 중분류 제목에 `(공통사업)` 또는 `(자체사업)`이 붙어 있다. `backfill_major_category_from_medium`을 적용해 괄호 안의 사업 구분을 대분류로 역채움하고 중분류에는 주제명만 남겼다.

총괄 행인 `공통사업+자체사업`, `공통사업 (저출산+고령사회)`은 광주에 한해 세부사업에서 제외했다.
역채움이 끝난 뒤 대분류를 `Ⅰ. 공통사업`, `Ⅱ. 지자체 자체사업`으로 표준화했다.

- 공통사업 leaf: 84건
- 자체사업 leaf: 194건
- 대분류 결측: 0건

### 3.2 경기

경기 원본행 3161~3175는 전체 예산을 주제와 사업 유형별로 다시 보여주는 요약 블록이다. 실제 사업 목록은 원본행 3176의 `1. 저출산 대책(공통사업)`부터 시작한다.

대분류 전파 기준인 원본행 3165 `Ⅰ. 공통사업`은 유지하고, 원본행 3162~3175의 나머지 요약 행을 세부사업에서 제외했다.

- 제외한 요약 행: 12건
- 최종 leaf: 126건
- 공통사업 leaf: 54건
- 자체사업 leaf: 72건
- 대분류·중분류 결측: 0건

### 3.3 대전

대전은 다른 지역과 계층 표기의 의미가 다르다.

- `[공 통 사 업]`, `[자 체 사 업]`: 대분류
- 로마숫자 제목: 중분류
- `(N개 과제)`로 끝나는 나머지 제목: 하위 소계
- 그 외 사업 행: 세부사업

하위 소계는 최종 분석 스키마에 필요하지 않으므로 별도 차원을 만들지 않고 기존 소계 행과 같이 세부사업에서 제외했다.
대괄호 제목에서 얻은 `공통사업`, `자체사업`은 최종 단계에서 `Ⅰ. 공통사업`, `Ⅱ. 지자체 자체사업`으로 표준화했다.

- 공통사업 leaf: 88건
- 자체사업 leaf: 142건
- 총 leaf: 230건
- 대분류·중분류 결측: 0건
- `(N개 과제)` 제목의 leaf 잔존: 0건

### 3.4 울산

울산은 계층 분류 자체는 정상이다. 다만 `1. 저출산 대책(공통사업)`에서 원본 소계와 leaf 합계의 차이가 크게 나타난다.

- 원본 소계: 60,111
- leaf 합계: 217,786
- 차이: 157,675
- 오차율: 262.31%

`보육비 전액 지원대상 확대` 157,490은 원본에서 국비 89,803과 지방비 67,687의 합계로 확인되며 실제 세부사업이다. 원본 소계와 맞지 않는다는 이유만으로 제거하지 않고 leaf 합계에 포함했다. 동일한 이름의 `청소년 방과후 아카데미 확대 및 내실화` 2건도 원본 확인 결과 정상적인 별도 사업이므로 모두 유지했다.

## 4. QA 개선

2016년 자료는 중분류 제목과 실제 소계 숫자가 서로 다른 행에 있는 경우가 많다. 기존 QA는 `중분류_소계` 제목 행에서만 예산을 읽어 대부분의 원본 소계를 결측 처리했다.

`build_subtotal_qa`를 확장해 다음 순서로 원본 소계를 선택하도록 수정했다.

1. `세부사업명 == "소계"`인 별도 숫자 행을 우선 사용
2. 별도 소계가 없으면 중분류 제목 행의 예산 사용
3. leaf 합계와 비교해 차이와 오차율 산출
4. 한쪽 값이 없으면 `불일치`가 아니라 `판정불가` 처리

QA 출력 컬럼은 다음과 같다.

- 지역, 대분류, 중분류
- 원본 소계값, 원본 소계 출처
- leaf 합계
- 차이, 절대차이
- 오차율, 절대오차율
- QA 병합상태, 결과, 허용기준결과

개선 후 실제 2016 데이터의 QA 현황은 다음과 같다.

- 전체 QA 그룹: 95개
- 원본 소계 숫자 확보: 93개
- 일치: 51개
- 불일치: 42개
- 판정불가: 2개
- 별도 소계 행 사용: 84개
- 중분류 제목 행 사용: 11개
- 오차율 허용: 84개
- 오차율 초과: 7개
- 허용기준 판정불가: 4개

재정대응지수에는 leaf 합계를 사용하며, 불일치 그룹은 차이 금액과 오차율을 별도 품질 정보로 제공한다.

## 8. 정제 결과 추가 이상 점검

시도별 정제 CSV 17개, 총 4,494행을 대상으로 계층 결측, 소계성 이름, 원본행 중복, 완전 중복, 예산 재계산을 점검했다.

### 8.1 수정 필요

- 광주 원본행 1913, 1914가 동일한 `국공립대 여성교수 및 여성교장, 교감 임용 확대` 사업으로 2행 남아 있다. 원본행 1913의 재원구분 `계\n(비예산)`을 총계 토큰으로 인식하지 못해, 원본행 1914~1915의 국비·지방비 합산 대표 행과 동시에 통과한 것이 원인이다. `계(비예산)` 변형을 총계로 정규화한 뒤 한 행만 유지해야 한다. 예산은 모두 0이어서 금액 합계 영향은 없지만 leaf 건수에는 영향을 준다.

### 8.2 원본 확인 결과 유지

- 대구 원본행 1095, 1096은 `공통사업 합계`의 국비·지방비 상세 행으로 확인해 `헤더반복` 처리했으며 최종 leaf에서 제외했다. 대구 leaf는 160건이고 중분류 결측은 0건이다.
- 부산 `출산 기념 축하 미역 지원` 원본행 791, 792, 798은 확인 결과 자동 중복 삭제 대상이 아니므로 모두 유지했다.
- 같은 지역·머리글·사업명이 반복되는 경우에도 연속 재원 블록을 기준으로 서로 다른 사업을 보존했다. 이 변경으로 이전의 이름 기반 합산에서 합쳐졌던 정상 사업을 복원했다.

### 8.3 허용 및 검토 사항

- 당해·전년도 예산이 모두 결측인 leaf 44건은 프로젝트 기준상 허용한다. 실제 정책성·비예산 사업일 수 있으므로 이상 후보에서 제외하고 원본 상태를 유지한다.
- 제주 원본행 7725의 `제주형주거복지종합계획수 립및추진 -노년층주택공급-`에는 사업명 내부 OCR 공백이 남아 있다. 현재 LLM은 `주요내용`만 정제하므로 사업명 정제 규칙에서 별도로 처리할 수 있다.
