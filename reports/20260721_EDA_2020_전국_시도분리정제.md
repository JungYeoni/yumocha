# 2020년도 전국 시도 세부사업 예산현황 분리·정제

## 1. 입력과 연도 설정

- 입력 파일: `세부사업표추출_2020년도 지방자치단체 저출산고령사회 시행계획 (제3차 기본계획)_칼럼정렬.xlsx`
- 입력 시트: `정리본_자동`
- 입력 규모: 7,003행, 12개 컬럼, 17개 시도
- 예산 컬럼: `2020년 예산`, `2019년 예산`
- 0원 토큰: `-`
- 단위표기 머리글: `UNIT_NOTATION_PATTERN` 적용
- 재원구분 정리: `NEEDS_FUNDING_SOURCE_CLEANUP=False`

`사업분류재정구분`의 비결측값은 `공통`, `자체`뿐이다. 계·국비·지방비 구조가 없으므로 재원구분 합산 유틸은 적용하지 않았다.

## 2. 2020년 행 유형 보정

2020년 원본의 `(공통사업)/(자체사업)` 중분류와 연도 고유 예외를 노트북에서 명시적으로 보정했다. 공통 QA 모듈에는 원본 소계를 보존하면서 별도 보정액으로 비교할 수 있는 선택적 인자를 추가했다. 이 인자를 쓰지 않는 다른 연도의 기본 동작과 출력 스키마는 바뀌지 않는다.

최종 행 유형은 다음과 같다.

| 행 유형 | 행 수 |
|---|---:|
| 세부사업 | 6,858 |
| 중분류 소계 | 100 |
| 대분류 소계 | 34 |
| 머리글 반복·제외 | 11 |

제외 11건은 서울 단위표기 1건, 경기 도·시/도비·시비 보조 소계 6건, 인천·세종·경기 전체 총계 3건, 경남 원본행 7,494의 잘못된 중분류 머리글 1건이다.

경남 원본행 7,494는 `3. 인구구조 변화 적극 대비 (자체사업)`로 적혀 있지만 뒤에 노후사업이 이어지고 실제 3번 소계가 원본행 7,651에 다시 나온다. 반복된 원본 오기 머리글로 제외했다.

원본행 7,494부터 실제 3번 소계인 7,651까지 행 단위 검증표를 만들었다. 오기 머리글 1행, 뒤따르는 노후사업 leaf, 실제 3번 소계까지 총 142행의 기대 행 유형·기대 중분류와 실제 라벨이 모두 일치했다.

## 3. 중분류 소계 QA

- QA 그룹: 100개
- 병합 상태: 100개 모두 양쪽 존재
- 엄격 일치: 67개
- 엄격 불일치: 33개
- 허용범위 내 불일치: 31개
- 허용기준 `허용`: 98개
- 허용기준 `초과`: 2개

| 지역 | 중분류 | 원본 소계 | 보정액 | QA 비교 소계 | leaf 합계 | 차이 | 판정 |
|---|---|---:|---:|---:|---:|---:|---|
| 경남 | 2. 함께 만들어가는 행복한 노후(자체사업) | 87,048 | -29,994 | 57,054 | 57,055 | +1 | 허용범위 내 불일치 |
| 울산 | 1. 함께 돌보고 함께 일하는 사회(공통사업) | 326,481 | 0 | 326,481 | 410,058 | +83,577 | 초과 |
| 울산 | 3. 인구구조 변화 적극 대비 (자체사업) | 283 | 0 | 283 | 333 | +50 | 초과 |

경남 원본 소계 87,048은 수정하지 않았다. 원본행 7,494에 잘못 포함된 29,994백만원만 `보정액`과 `보정근거`에 기록하고, 차이·오차율·허용 판정은 57,054를 기준으로 계산했다. 따라서 leaf 57,055와의 차이는 +1이며 허용범위 내 불일치다. 초과 항목은 기존 울산 2건뿐이다.

## 4. 텍스트 정제와 LLM 상태

- 세부사업명·주요내용 공백 및 선행 불릿 정리 완료
- 정규식 기반 지원대상·지원내용 분리 완료
- 유사 사업명 검토 후보: 250건
- LLM 대상: 주요내용 비결측 6,849건
- 최종 체크포인트: 6,858행
- 최종 상태: 성공 6,828건, 보존위반 원문 유지 21건, 결측 9건, 실패 0건

실제 Upstage API로 보존형 교정을 실행했다. 체크포인트는 인덱스뿐 아니라 `지역`, `원본행`, `세부사업명`을 함께 대조하고, `needs_llm_rerun()`에 걸리거나 실패 상태인 행만 다시 호출한다. 20행 단위 중간 저장과 API 재시도, 호출 간격 제한을 적용했다.

- 마지막 완료 재개 실행: 기존 완료 6,770건 재사용, 신규 호출·성공 88건, 실패·보류 0건
- 완료 후 전체 재실행: 기존 완료 6,858건 재사용, 신규 호출 0건, 실패 0건
- 숫자·금액·퍼센트·범위·추출 가능한 고유명사 보존 위반 응답 21건은 원문 유지
- 최종 산출물 보존 검사: 6,858건 통과, 위반 0건
- 주요내용 결측 9건은 API를 호출하지 않고 결측 유지

## 5. 비-LLM 산출물 검증

- wide: 17개 파일, 6,858행, 공통 15개 컬럼
- long: 17개 파일, 13,716행, 15개 컬럼
- 필터링 전 전체 원본: 17개 파일, 7,003행
- QA·행 유형·라벨 전파·LLM 요약·보존위반 검토표 포함 총 56개 CSV가 UTF-8-SIG
- wide·long 저장 후 재로딩 행 수와 스키마 일치
- 빈 정제 결과 0건, wide·long 키 중복 0건, 인코딩 대체 문자 0건
- 완료 체크포인트 전체 재사용 실행 뒤 실행요약을 제외한 55개 CSV의 SHA-256 변화 0건
- 원본 XLSX SHA-256 `8F05879D7BE6634DCBEEDB7CF571840DB80F126942F65D063AC40552A9803068` 유지

코드 검증 결과:

```text

RUN_LLM=False 사전조건 전체 실행: 통과, 실행 오류 0건
RUN_LLM=True 실제 정제 전체 실행: 통과, 실행 오류 0건
완료 체크포인트 재사용 전체 실행: 통과, 신규 호출 0건
```

`jupyter nbconvert --execute --inplace`로 노트북을 처음부터 끝까지 실행했다. 샌드박스 내부 네트워크 제한은 초기 청크 전건 연결 실패로 조기 감지했고, 네트워크 승인 실행에서 체크포인트를 재개했다. 초기 동시성에서 발생한 일시적 rate limit은 성공 행을 재사용한 채 호출 속도를 낮춰 해결했다. API 키 값이나 오류 응답 본문은 출력·산출물·커밋에 기록하지 않았다.
