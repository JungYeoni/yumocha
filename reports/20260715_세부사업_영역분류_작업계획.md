## 1. 배경

재정팀 로드맵은 사업 범위를 Level 1~3로 구분하며, 각 레벨은 다음 항목의 트레이드오프로 구성된다: 컴퓨터 분류 도움의 필요성, 연구자 라벨링 노력의 실현 가능성, 단년도 회귀분석의 중요도, 그리고 목표로 하는 예산 대응·상관관계 수준과 대상 연도 수.

| 구분 | Level 2 (채택) | Level 3 (참고) |
|---|---|---|
| 대상 연도 수 | 5개년 | 10개년 |
| 컴퓨터 분류 도움 | 있으면 유의미하나 학습데이터 부족 우려 | 필요 |
| 연구자 라벨링 | 감당 가능한 수준 | 노가다 수준으로 불가능 |
| 단년도 회귀분석 중요도 | mid | low |
| 목표 수준 | 예산 대응 mid / 상관관계 mid~low | 예산 대응 high / 상관관계 low |

팀은 **Level 2로 진행하기로 결정**했다. Level 2의 대상 범위는 `세부사업표추출_...` 파일 기준 **2021년도부터**이며, 제4차 기본계획 기간(2021~2025)에 해당한다.

**현재 데이터 확보 상태**: `data/raw/`에는 2024년도까지만 원본 파일이 있고, **2025년도 파일은 아직 확보되지 않았다.** `data/raw/칼럼정렬/2016-2025_표칼럼_범용정렬_폴더오류수정.bas`라는 파일명에 "2025"가 포함되어 있어 추후 2025년도 파일 처리를 염두에 둔 것으로 보이나, 실제 데이터는 미확보 상태다.

**라벨링 현황**: 재정팀이 직접 라벨링(대분류·중분류)한 연도는 **2021년 1개년뿐**이다. 당초 재정팀 자료는 제3차(2016)·제4차(2021) 두 시작연도를 라벨링 기준으로 제안했으나, Level 2 범위가 2021년부터이므로 2016년 라벨은 이번 범위에 포함되지 않는다. 이는 로드맵에 명시된 "학습데이터 부족" 우려와 직접 연결된다 — 학습 가능한 라벨 데이터가 1개년뿐이라는 뜻이기 때문이다.

**분류 대상 taxonomy(재정팀 제공, 예시)**: 대분류 5개(경제·고용·주거 / 가족·양육 / 보건·의료 / 시민·제도 / 기타) 아래 각각 2~3개의 중분류로 구성된다.

**공통 선행 작업**: 세부사업명 문장 정렬(띄어쓰기, 오탈자 수정 등)이 재정팀 자료에 선행 조건으로 명시되어 있으며, 이는 `20260712_시도별_예산현황_분리정제_작업계획.md`의 `정리본_자동` 텍스트 정제 작업과 동일한 문제다.

`정리본_자동` 기준 행수는 다음과 같다(대분류_소계·중분류_소계·노이즈 행 포함).

| 연도 | 행수 | 상태 |
|---|---|---|
| 2021 | 8,115 | 라벨링 완료(재정팀) — 학습 데이터 |
| 2022 | 8,472 | 미분류 — 분류 대상 |
| 2023 | 7,712 | 미분류 — 분류 대상 |
| 2024 | 7,340 | 미분류 — 분류 대상 |
| 2025 | - | 원본 파일 미확보 |

- 학습 데이터(2021년 1개년): 약 8,000행. 소계·노이즈 행을 제외한 실제 세부사업(leaf)만 남기면 이보다 적다(2024년 서울 표본 기준 leaf 비율이 전체 행의 약 80% 수준이었던 것을 참고하면, 학습 가능한 leaf 행은 6,000행 내외로 추정된다). 당초 2개년(2016+2021) 라벨을 가정했을 때보다 학습 데이터가 절반 수준으로 줄어, 분류기 성능이 기대에 못 미칠 가능성을 열어두어야 한다.
- 분류 대상(2022~2024, 2025 확보 시 추가): 약 2만 3,000행.
- 데이터 규모 자체는 TF-IDF 기반 텍스트 벡터화에 부담이 없다. 다만 위 학습 데이터 부족 우려는 벡터화 속도의 문제가 아니라 **분류기의 일반화 성능** 문제이므로, §3-2에서 성능을 먼저 확인하고 다음 단계 진행 여부를 결정한다.

## 3. 제안 파이프라인

컴퓨터 분류 단계를 3단계로 나누고, 앞 단계에서 검증한 결과가 충분하지 않을 때만 다음 단계로 진행한다. 학습 데이터가 2021년 1개년뿐이므로, **2단계(TF-IDF) 성능을 먼저 확인한 뒤 다음 단계 진행 여부를 결정**하는 것으로 팀 합의가 있었다.

### 3-1. 1단계 — 키워드 사전 기반 규칙 매칭

재정팀이 제공한 중분류별 키워드(예: 청년고용·창업지원, 주거안정, 출산장려금, 육아인프라, 산후조리, 워라밸 등)로 사전을 구성하고, `세부사업명`+`주요내용`에 해당 키워드가 있으면 그 중분류로 태깅한다.

- 장점: 로직이 투명해 분류 근거를 그대로 설명할 수 있고, 학습 데이터 부족의 영향을 받지 않는다.
- 한계: 사전에 없는 표현(동의어, 신조어)은 분류하지 못한다.

### 3-2. 2단계 — TF-IDF + 계층적 분류기 (우선 시도, 성능에 따라 채택 여부 결정)

1. **피처화**: `세부사업명`(+`주요내용` 보조)을 형태소 분석 후 TF-IDF 벡터로 변환
2. **분류기 구조**: 대분류를 먼저 분류하고, 그 안에서 중분류를 분류하는 계층적 분류(대분류 분류기 → 중분류 분류기). 대분류가 5개로 적어 단일 다중분류보다 안정적일 것으로 예상
3. **학습/검증**: 2021년 라벨 데이터를 train/validation으로 나눠 교차검증(`random_state=42` 고정)해 이 방법의 실효성을 확인. **이 단계의 성능 결과를 기준으로 2단계를 본 파이프라인으로 채택할지, 1단계(키워드)+연구자 검수 중심으로 전환할지 결정**
4. **적용**: 성능이 확인되면 2022~2024년(및 확보 시 2025년)에 예측하고, 확신도(`predict_proba`)를 함께 저장

### 3-3. 3단계 — 저확신 항목 LLM 보조 분류 (2단계 채택 시에만 진행)

2단계에서 확신도가 낮게 나온 항목만 선별해, 대분류/중분류 정의와 2021년 예시를 프롬프트에 포함해 LLM(Claude)으로 분류를 보조한다. 전체 데이터를 LLM으로 처리하면 비용·시간이 많이 들기 때문에 저확신 항목으로 범위를 한정한다.

### 3-4. 검수 루프

1. 자동분류(1·2단계) 결과 중 확신도가 낮은 항목을 우선순위로 연구자가 검수
2. 검수 완료 라벨을 학습 데이터에 반영해 필요 시 재학습
3. 남은 저확신 항목을 재분류
4. 대상 연도(2022~2024, 확보 시 2025) 라벨 확정

### 방법별 소요 시간 비교

| 단계 | 상대 속도 | 비고 |
|---|---|---|
| 1단계 키워드 매칭 | 가장 빠름 | 단순 문자열 포함 검사 |
| 2단계 TF-IDF + 분류기 | 빠름 | 형태소 분석이 상대적 병목이나 분 단위로 예상 |
| 3단계 LLM 보조 | 가장 느림 | 저확신 항목만 선별해 범위를 최소화 |

## 4. 기술 스택

- **형태소 분석**: KoNLPy(Okt) 또는 Mecab — 처리 속도 우선이면 Mecab 권장
- **벡터화/분류기**: scikit-learn `TfidfVectorizer`, `LogisticRegression` 또는 `LightGBM` (프로젝트 기본 스택과 일치)
- **LLM 보조(3단계, 조건부)**: Claude API, 저확신 항목 한정

## 5. 방법론적 유의사항

- **데이터 누수 방지**: TF-IDF 벡터라이저와 분류기는 2021년(라벨 데이터)에서만 `fit`하고, 나머지 연도는 `transform`/`predict`만 적용한다.
- **학습 데이터 부족 리스크**: 라벨 데이터가 1개년(2021)뿐이라 일반화 성능이 낮게 나올 수 있다. 교차검증 결과가 §6의 성능 기준 미만이면 2단계를 채택하지 않고 1단계+연구자 검수 중심으로 전환한다.
- **선행 전처리**: 세부사업명 문장 정렬(띄어쓰기·오탈자 정리)이 분류 정확도에 직접 영향을 준다. `20260712_시도별_예산현황_분리정제_작업계획.md`의 텍스트 정제 로직 재사용을 검토한다.
- **클래스 불균형**: 대분류 5개 중 "기타"로 쏠릴 가능성이 있어, 학습 전 라벨 분포를 먼저 확인한다.
- **평가지표**: 2021년 교차검증 결과를 정확도뿐 아니라 대분류별/중분류별 F1-score로 보고해, 특정 영역에 편향된 성능인지 확인한다.
- **재현성**: `random_state=42` 고정, 학습/검증 분할 기준을 코드 상단에 명시한다.

## 6. 결정 사항

**확정:**

| 항목 | 결정 |
|---|---|
| 로드맵 레벨 | Level 2 채택 |
| 대상 연도 범위 | 2021~2025 (제4차 기본계획 기간), 2025는 원본 미확보로 보류 |
| 학습 라벨 데이터 | 2021년 1개년 (재정팀 라벨링분) |
| 분류 방법 순서 | 1단계(키워드) 우선 적용 → 2단계(TF-IDF) 시도 후 성능 확인 → 성능 기준 충족 시에만 3단계(LLM 보조) 진행 |
| 2단계 채택 기준(성능 임계값) | 대분류 macro F1 ≥ 0.7, 중분류 macro F1 ≥ 0.55. 미달 시 2단계 미채택, 1단계(키워드)+연구자 검수 중심으로 전환. accuracy 대신 macro F1을 쓰는 이유는 클래스 불균형("기타" 쏠림) 때문이며, 절대값과 함께 최빈 클래스 baseline 대비 개선폭도 같이 보고 |

**결정 필요:**

1. 대분류 5개·중분류 taxonomy를 재정팀 제공안 그대로 확정할지, 데이터 확인 후 조정할지
2. 2단계 채택/미채택을 가르는 성능 기준(F1-score 등 구체적 임계값)
3. 2021년 라벨 데이터(재정팀 라벨링분)의 공유 형식과 시점
4. 세부사업명 문장 정렬(전처리) 작업을 이 작업 전에 완료할지, 병행할지
5. 2025년도 원본 파일 확보 일정
