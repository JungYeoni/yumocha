# 시행계획 세부사업 예산현황 — 시도별 분리·정제 작업계획 — 2026-07-12

작성: Claude Code / 대상: 유모차팀  
원본 파일을 직접 열어 확인한 실제 구조를 반영해 작성함.

---

## 1. 배경

- `data/raw/`에는 **연도별 파일 9개**(2016~2024, 제3차/제4차 기본계획)가 있고, 시도별 파일 분리는 없음.
- 파일마다 시트는 `Table 1` 하나뿐이며, 그 안에 **17개 시도가 정해진 순서로 이어 붙어 있는 단일 롱 테이블**이다 (총 7,844~13,014행).
- 시도 순서는 고정: 서울 → 부산 → 대구 → 인천 → 광주 → 대전 → 울산 → 세종 → 경기 → 강원 → 충북 → 충남 → 전북 → 전남 → 경북 → 경남 → 제주.
- 소관기관(교육청) 정보는 시도 타이틀 행에 `붙임 (OO시 OO시교육청)` 형태로 같이 붙어 있고, 별도 컬럼으로 존재하지 않는다.

## 2. 주요 발견

### 2-1. 시도 경계 타이틀: 텍스트 vs 이미지 (+ OCR 검증)

같은 형식의 파일이라도 연도에 따라 시도 구분 타이틀이 **텍스트로 존재하는 경우**와 **이미지로만 존재하는 경우**가 섞여 있다.

| 연도 | 총 행수 | 임베디드 이미지 수 | 시도 타이틀 텍스트 검출                    | 비고                                                                                  |
| ---- | ------- | ------------------ | ------------------------------------------ | ------------------------------------------------------------------------------------- |
| 2016 | 7,844   | 314                | 11/17 (서울·대구·광주·울산·세종·충남 누락) | 이미지가 본문 셀에도 다수 섞여 있어 타이틀 이미지만 특정하기 어려움                   |
| 2017 | 13,014  | 333                | 13/17 (서울·대구·인천·대전·세종·충남 누락) | 상동                                                                                  |
| 2018 | 9,301   | 829                | **17/17**                                  | 이미지 개수는 많지만 타이틀은 텍스트로 존재                                           |
| 2019 | 9,804   | **17**             | 12/17 (서울·대구·인천·세종·충남 누락)      | 이미지 17개 = 시도 수 17개, 위치가 섹션 시작과 정확히 일치 → **타이틀이 전부 이미지** |
| 2020 | 7,904   | **17**             | 13/17                                      | 2019와 동일 패턴                                                                      |
| 2021 | 10,672  | 468                | **17/17**                                  |                                                                                       |
| 2022 | 9,751   | 181                | **17/17**                                  |                                                                                       |
| 2023 | 8,723   | **17**             | 12/17 (서울·대구·광주·울산·세종 누락)      | 2019/2020과 동일 패턴                                                                 |
| 2024 | 8,234   | 87                 | **17/17**                                  | `붙임 (OO시 OO시교육청)` 정규식으로 전부 매칭, 구조 가장 깔끔                         |

**이미지 내용은 openpyxl의 셀 텍스트 추출로는 읽을 수 없지만, OCR을 돌리면 코드로 완전히 자동화할 수 있다.** 2016년 파일에서 텍스트로 누락됐던 6개 전부를 `tesseract` + `kor` 언어팩으로 검증한 결과는 다음과 같다.

| 시도           | 확인된 행 | OCR 원본 결과                | 17개 후보 매칭                                 |
| -------------- | --------- | ---------------------------- | ---------------------------------------------- |
| 서울특별시     | ~61       | `서울특별시`                 | 일치                                           |
| 대구광역시     | 1,115     | (육안 확인, OCR 재검증 생략) | 일치                                           |
| 광주광역시     | 1,660     | `광주광역시`                 | 일치                                           |
| 울산광역시     | 2,686     | `올산광역시` (한 글자 오독)  | 편집거리 최소값으로 자동 보정 → **울산광역시** |
| 세종특별자치시 | 2,994     | `세종특별자치시`             | 일치                                           |
| 충청남도       | 4,594     | `충청남도`                   | 일치                                           |

6개 중 5개는 OCR이 정확히 맞혔고, 1개(울산)는 한 글자를 오독했다. 다만 **정답이 17개 시도명으로 정해진 폐쇄형(closed-set) 문제**이므로, OCR 결과와 17개 후보 사이 편집거리(fuzzy match)를 계산해 가장 가까운 후보로 자동 보정하면 사실상 100% 정확도를 얻을 수 있다. 전부 표준 행정구역 순서와도 일치했다.

> 참고: 이미지 안의 회색 반복 이미지 대부분(2016년 314개 중 다수)은 실제로는 **"Bridge Plan 2020" 워터마크와 "2016년 지방자치단체 시행계획" 페이지 헤더**였다. PDF→엑셀 변환 과정에서 매 페이지마다 삽입된 장식 이미지로, 항상 1번째 열(빈 칸)에만 붙어 있어 데이터 텍스트와 안 겹치고 텍스트 읽기에 영향이 없다. 진짜 시도명 타이틀 배너는 이 워터마크와 크기가 달라 구분 가능했다.

**자동화 파이프라인 (코드로 완결, 팀원 누구나 실행 가능 — Claude Code 불필요):**

1. 파일별 임베디드 이미지의 바이트 크기를 세어 최빈 크기(워터마크)를 제외
2. 남은 후보 중 각 시도 섹션 시작 지점(텍스트로 찾은 앞뒤 경계 사이의 빈 구간) 근처 이미지를 추출
3. `pytesseract`로 OCR 실행
4. OCR 결과를 17개 시도명 후보와 편집거리로 비교해 가장 가까운 후보로 확정 (`rapidfuzz` 사용)
5. 편집거리가 임계값 이상으로 먼 경우에만 사람 검수 플래그

**필요 환경**: 시스템에 `tesseract`(Homebrew: `brew install tesseract tesseract-lang`, Linux: `apt install tesseract-ocr tesseract-ocr-kor`) + Python `pytesseract`, `rapidfuzz` 패키지 (프로젝트 의존성에 추가 완료).

### 2-2. 헤더/컬럼 구조가 연도마다 다름

> **`skiprows`란**: `pandas.read_excel()`이 표를 읽을 때 맨 위에서 몇 줄을 건너뛸지 지정하는 인자다. 이 원본 파일들은 실제 헤더(`세부사업명`, `예산` 등) 위에 "붙임 (OO시 OO교육청)", "(단위: 백만원)" 같은 타이틀/부제목 줄이 붙어 있는데, 이 타이틀 줄 개수가 연도마다 달라서 `skiprows` 값도 파일마다 달라진다. 아래 표의 `skiprows`는 "실제 헤더가 시작되기 전까지 건너뛰어야 하는 줄 수"를 뜻한다.

| 연도      | skiprows | 헤더 행수 | 특이사항                                                                                                                                                       |
| --------- | -------- | --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 2016~2019 | 1        | 2행       | `재정 구분` 컬럼에 `계/국비/지방비`가 있어 **세부사업 1개당 3행**으로 분리됨. `공통/자체` 컬럼 없음. `classify_row`/`parse_budget_sheet` 로직이 그대로 안 맞음 |
| 2020      | 2        | 2행       | §3-4 참조 함수와 거의 일치                                                                                                                                     |
| 2021      | 3        | 2행       | 타이틀 위에 부제목 줄이 하나 더 있어 skiprows가 밀림                                                                                                           |
| 2022      | 1        | 2행       |                                                                                                                                                                |
| 2023      | 3        | 2행       | 컬럼명 자체에 시도명이 붙는 경우 있음(예: `세부사업명 (서울)`) → 컬럼명 정규화 로직 필요                                                                       |
| 2024      | 2        | 2행       | 참조 함수 가정과 정확히 일치                                                                                                                                   |

**`skiprows`는 파일마다 다르므로 고정값을 쓸 수 없고, 파일별로 자동 탐지하는 로직이 필요하다.**

### 2-3. (a)/(b) 연도 라벨 자동판별

`find_budget_columns` 로직(컬럼명에서 4자리 연도를 뽑아 큰 값=당해년도, 작은 값=전년도로 판별)은 그대로 유효함을 확인했다. 2016~2019년 파일은 `(A)/(B)` 라벨, 2022~2024년은 `(a)/(b)` 라벨을 쓰지만 둘 다 이 로직으로 커버된다.

## 3. 처리 파이프라인

### 3-1. 범위 (Scope)

"1개 연도, 그 연도의 여러 시도" 시나리오는 **어느 연도를 골라도 실제로 가능**함을 확인했다 (파일 하나에 이미 17개 시도가 다 들어있음). 다만 구조적으로 두 그룹으로 나뉘므로, 검증용 샘플은 아래 3케이스로 진행할 것을 제안한다.

1. **2024년 · 서울/부산** — 텍스트 타이틀 명확, 헤더 구조도 참조함수와 가장 근접 → 기본 파이프라인 검증
2. **2019년 · 서울** — 타이틀이 이미지뿐인 케이스 → OCR 기반 시도 매핑 로직 검증
3. (선택) **2016년 또는 2017년 · 임의 1개 시도** — 계/국비/지방비 3행 분리 구조 검증 (별도 파싱 함수 필요하므로 이번 1차 샘플에서는 제외하고 2차로 미루는 것을 권장)

### 3-2. 실행 순서

OCR은 **전체 데이터를 텍스트화하는 단계가 아니라, 시도 경계 타이틀(17개)만 확정하는 데 쓰인다.** 예산 숫자·세부사업명 등 본문은 이미 텍스트라 pandas로 바로 읽으면 되고 OCR이 필요 없다 (2018·2021·2022·2024는 경계 타이틀도 텍스트라 OCR 자체가 아예 불필요). 정제(계층 태깅, ffill 등)보다 **시도별 슬라이싱을 먼저** 해야, 정제 로직이 시도 경계를 넘어 오염되는 것을 막을 수 있다.

1. **skiprows 자동탐지** — 파일별 헤더 시작 행 확인
2. **시도 경계 찾기** — 텍스트 정규식 우선, 텍스트 없는 구간만 OCR + 17개 후보 편집거리 매칭으로 보완
3. **시도별 슬라이싱** — 확정된 경계 기준으로 시트를 시도별 원본 조각(raw slice)으로 분리
4. **조각별 정제** — 헤더 평탄화 → 계층 태깅 → 대분류/중분류 ffill → leaf/long 변환 (§3-4 참조 함수 재사용)
5. **QA 대조** — 시도별로 소계 행 vs leaf 합계 검증
6. **저장** — `data/processed/`에 wide·long, `data/interim/`에 경계탐지로그·소계포함 원본, `reports/`에 QA 결과

### 3-3. 신규 필요 함수

기존 요구사항(헤더 평탄화, 연도 자동판별, 계층 태깅, 대분류/중분류 분리·ffill, 메타컬럼 추가, leaf 추출, long 변환, QA 대조)은 그대로 유지한다. 다만 아래 항목이 **선행 작업으로 추가**되어야 한다.

1. **`detect_header_skiprows(filepath)`**: 파일별로 타이틀 행 개수를 자동 탐지 (현재 1/2/3 세 가지 값이 혼재)
2. **`find_sido_boundaries(filepath)`**: 시트 전체를 스캔해 17개 시도 섹션의 시작 행을 찾는 함수
   - 텍스트 타이틀이 있으면 정규식(`붙임\s*\(([^)]+)\)` 등, 공백 스트립 후 매칭)으로 우선 추출
   - 텍스트가 없는 구간은 임베디드 이미지 중 워터마크(최빈 바이트 크기)를 제외한 후보를 anchor 행 기준으로 추려서, **`pytesseract` OCR + 17개 시도명 후보 편집거리 매칭**으로 자동 확정 (§2-1 참조). 순번 추정은 OCR도 실패할 때만 최후 수단으로 사용
   - 판독 결과는 `data/interim/{연도}_시도경계_탐지로그.csv`에 "텍스트/OCR/순번추정" 중 어떤 방식으로 확정했는지, OCR인 경우 편집거리 값까지 기록
3. **소관기관 파싱**: 타이틀 텍스트 `OO시 OO시교육청` 형태에서 시도명과 소관기관을 분리하는 정규식 필요 (텍스트가 없는 이미지 케이스는 소관기관을 `None` 또는 `미확인`으로 두고 별도 표시)
4. **컬럼명 정규화**: 2023년처럼 컬럼명에 시도명이 붙는 경우 `세부사업명` 등 표준 컬럼명으로 리네이밍 후 처리
5. **(2차 범위) 2016~2019 전용 파서**: `계/국비/지방비` 3행 구조를 어떻게 다룰지는 정책적 결정이 필요함 (§6 참조) — 이번 1차 샘플(2024·2019)에는 해당 없음

### 3-4. 기존 함수 재사용 방침

`flatten_columns`, `find_budget_columns`, `classify_row`, `parse_budget_sheet`, `to_leaf`, `to_long`, `qa_check`는 **2020·2022·2024년처럼 표준 구조인 파일**에는 그대로 재사용 가능하다. 다만:

- `parse_budget_sheet`의 `skiprows=2` 고정 인자는 `detect_header_skiprows` 결과값으로 대체
- 시도별로 시트를 나누는 전처리 단계(`find_sido_boundaries`로 슬라이싱 후 시도별 `DataFrame`으로 분리)가 `parse_budget_sheet` 호출 **이전**에 추가로 필요
- `qa_check`의 중분류 소계 매칭 로직은 원본 그대로 사용 가능하나, 시도 단위로 나눠서 실행해야 함

## 4. 산출물 (Deliverables)

이 저장소는 `README.md`에 정의된 `data/raw/` → `data/interim/` → `data/processed/` 3단 폴더 구조를 따른다. 이번 작업도 이 규칙을 지켜야 하며, 임의로 새 폴더를 만들거나 `data/raw/`에 있는 원본을 직접 수정/덮어쓰기 하지 않는다.

- **`data/raw/`**: 원본 엑셀 9개, **읽기 전용**으로만 사용 (git 추적 제외 대상이므로 이 안에서 아무것도 새로 만들지 않음)
- **`data/interim/`**: 시도 경계로 슬라이싱한 중간 산출물
  - 시도 경계 탐지 결과 요약표: `data/interim/{연도}_시도경계_탐지로그.csv` (텍스트/OCR/순번추정 중 어떤 방식으로 시도명을 확정했는지, OCR 편집거리 값과 함께 기록 — 편집거리가 큰 항목은 검수 대상으로 표시)
  - 소계 행 포함 시도별 wide 원본(QA용 보관): `data/interim/{연도}_{시도}_소계포함_raw.csv`
- **`data/processed/`**: 모델/분석 입력용 최종 정제 데이터
  - 시도별 wide(leaf) 정제 테이블: `data/processed/{연도}_{시도}_세부사업_정제.csv`
  - 시도별 long(tidy) 테이블: `data/processed/{연도}_{시도}_세부사업_long.csv`
- **`reports/`**: 팀 공유용 QA 결과와 이 계획 문서 같은 리포트류
  - QA 대조 결과: `reports/{연도}_{시도}_QA_불일치목록.csv` (불일치 없어도 "검증 완료" 표시로 빈 파일 생성)

위 산출물을 §3-1의 1·2번 케이스(2024 서울/부산, 2019 서울) 분량으로 우선 생성해 팀 공유. 코드는 CLAUDE.md의 역할 분리 규칙에 따라 `src/features/`(파싱·경계탐지·피처), `src/evaluation/`(QA 대조)로 나눠 작성한다.
