## 1. 작업 배경

2021·2022·2023·2024년도 시행계획 시도 분리·정제 노트북에는 행 분류, 계층 라벨 전파, 예산 숫자 변환, 텍스트 정리 등 동일하거나 유사한 함수가 반복해서 정의되어 있다. 이 구조에서는 같은 오류를 연도별 노트북에서 각각 수정해야 하고, 연도 확장 과정에서 과거 보완 사항이 누락될 가능성이 있다.

이에 공통 함수를 `src/features/` 아래의 모듈로 분리하고, 노트북은 연도와 원본 데이터의 예외만 인자로 전달하도록 리팩터링한다. 함수 구현은 한 곳에서 관리하되, 연도별 데이터 차이를 무조건 같은 규칙으로 처리하지 않는 것을 원칙으로 한다.

## 2. 모듈 구성 계획

```text
src/features/
├── pipeline_common.py  # 행 분류, 계층 전파, 숫자·텍스트 정제, 저장 경로
├── text_match.py       # 유사 사업명 비교와 지원대상·지원내용 추출
└── llm_refine.py       # LLM 보존형 교정과 숫자 보존 QA
```

현재 `pipeline_common.py`, `text_match.py`, `llm_refine.py`가 구현되어 있다.

## 3. `pipeline_common.py` 구현 내용

### 3-1. 시도별 저장 경로 생성

`get_sido_dir()`는 중간 산출물 루트와 지역명을 받아 `data/interim/{지역}` 형태의 경로를 생성한다. 노트북의 전역 경로에 의존하지 않도록 `interim_dir`을 명시적으로 전달한다.

### 3-2. 원본 행 유형 분류

`classify_row()`는 세부사업명에 따라 다음 네 유형을 반환한다.

- `대분류_소계`
- `중분류_소계`
- `헤더반복`
- `세부사업`

공통 판별 규칙에는 결측·빈 문자열, `세부사업명` 반복 머리글, `붙임(시도명)` 형태의 시도 제목, 로마 숫자로 시작하는 대분류, 숫자와 공통·자체 표기로 구성된 중분류가 포함된다.

특정 연도에만 나타나는 반복 머리글은 `extra_header_patterns` 인자로 추가할 수 있다. 예를 들어 2022년도 전용 반복 머리글을 공통 함수에 하드코딩하지 않고 호출부에서 전달할 수 있다.

### 3-3. 대분류·중분류 라벨 전파

`assign_labels()`는 지역별 데이터를 `원본행` 순서로 정렬한 뒤, 대분류 및 중분류 소계의 사업명을 뒤따르는 세부사업 행에 전방 채움한다.

필수 컬럼이 없으면 조용히 잘못된 결과를 만들지 않고 `KeyError`를 발생시킨다. 기본 컬럼명은 다음과 같으며 필요한 경우 인자로 변경할 수 있다.

- 원본 순서: `원본행`
- 행 유형: `사업행구분`
- 사업명: `세부사업명`

### 3-4. 텍스트 정제

`clean_text()`는 다음 작업을 수행한다.

- PUA(Private Use Area) 문자 제거
- 줄바꿈·탭·연속 공백을 일반 공백 한 칸으로 변환
- 문자열 앞뒤 공백 제거
- 선택적으로 문자열 맨 앞의 불릿 제거

세부사업명의 단어 사이 공백은 제거하지 않는다. 불릿 제거도 문장 중간의 하이픈이나 가운데점을 손상시키지 않도록 선행 불릿에만 적용한다.

`normalize_budget_type()`은 `공통 / 자체` 구분값의 불필요한 공백을 별도로 제거한다. 세부사업명과 공통·자체 구분값에 서로 다른 정제 기준을 적용하기 위한 함수다.

### 3-5. 예산 숫자 변환

`to_numeric_budget()`는 콤마를 제거하고 예산 문자열을 숫자로 변환한다. 실제 결측값은 0으로 바꾸지 않고 결측 상태를 유지한다.

0으로 해석할 원본 표기는 `zero_tokens` 인자로 전달한다.

```python
# 기본 규칙
to_numeric_budget(series, zero_tokens=("-",))

# 2023년도에서 별도 표기를 0으로 처리하기로 확정한 경우
to_numeric_budget(series, zero_tokens=("-", "·", "비예산"))
```

이 구조를 통해 `-`, `·`, `비예산`, 결측을 모든 연도에서 무조건 같은 의미로 처리하는 것을 방지한다.

### 3-6. 예산 증감 재계산

`calculate_budget_changes()`는 숫자형으로 변환된 당해예산과 전년도예산을 받아 `당해예산`, `전년도예산`, `증감액`, `증감율`을 생성한다. 증감액은 `당해예산 - 전년도예산`, 증감율은 `증감액 / 전년도예산 × 100`으로 계산한다.

전년도예산이 0이거나 어느 한쪽 예산이 결측이면 증감율은 결측으로 유지한다. 두 입력의 인덱스가 다르면 pandas의 묵시적 정렬로 잘못된 사업끼리 계산되는 것을 막기 위해 오류를 발생시킨다.

### 3-7. 원본 `Table 1` 대조

`show_table1_around()`는 원본 `Table 1`에서 특정 엑셀 행 주변을 DataFrame으로 반환한다. 노트북의 `df_table1` 전역변수에 의존하지 않고 원본 DataFrame을 인자로 받는다.

기본 컬럼 인덱스는 수정된 기준인 `(0, 2, 3)`을 사용한다. 연도별 원본 표 구조가 다르면 `column_indices`를 호출부에서 변경할 수 있다.

### 3-8. 중분류 소계 QA

`build_subtotal_qa()`는 원본 중분류 소계와 같은 지역·대분류·중분류에 속한 세부사업 예산 합계를 비교한다. 기본 허용오차는 0이며, 자료 단위나 반올림 특성에 따라 `tolerance`를 명시적으로 전달할 수 있다.

결과에는 `원본_소계값`, `leaf_합계`, `차이`, `오차율(%)`, `QA_병합상태`, `결과`가 포함된다. `오차율(%)`은 `(leaf 합계 - 원본 소계) / 원본 소계 × 100`으로 계산하며 소수점 둘째 자리까지 표시한다. 원본 소계가 0이거나 한쪽 값이 없어 비교할 수 없는 경우에는 오차율을 결측으로 유지한다. 원본 소계만 있거나 leaf 합계만 있는 그룹, 비교값이 결측인 그룹은 일치로 처리하지 않는다. 같은 그룹에 중분류 소계가 여러 행이면 잘못된 병합을 막기 위해 오류를 발생시킨다. QA 결과 CSV의 파일명과 저장 경로는 연도별 노트북에서 관리한다.

## 5. `llm_refine.py` 구현 내용

### 5-1. LLM 단건 호출

`call_llm_once()`는 노트북 전역변수 대신 `client`와 `llm_config`를 인자로 받아 API를 한 번 호출한다. JSON 응답에서 `정제문장`을 추출하며, JSON 파싱 실패·키 누락·문자열이 아닌 결과는 `None`으로 처리한다. 네트워크 오류 등 API 호출 자체의 예외는 재시도 함수에서 처리할 수 있도록 전달한다.

### 5-2. 재시도와 원문 폴백

`clean_sentence()`는 다음 규칙을 적용한다.

- 주요내용이 결측이면 API를 호출하지 않고 `None` 유지
- 기본 2회 시도(최초 호출 1회 + 재시도 1회)
- 정상 결과가 반환되면 즉시 종료
- 모든 시도가 실패하면 원문 유지

실제 API 함수는 `call_once` 인자로 주입한다. 이 구조를 통해 테스트에서는 외부 API를 호출하지 않고 성공·실패·재시도 동작을 검증할 수 있다.

노트북에서는 다음과 같이 설정한다.

```python
from functools import partial

from src.features.llm_refine import (
    call_llm_once,
    clean_sentence,
    extract_numbers,
    needs_llm_rerun,
    numbers_preserved,
)

call_once = partial(
    call_llm_once,
    client=client,
    llm_config=llm_cfg,
)

df_leaf_final["주요내용_정제"] = df_leaf_final.apply(
    lambda row: clean_sentence(
        row["세부사업명"],
        row["주요내용"],
        call_once=call_once,
    ),
    axis=1,
)
```

### 5-3. 숫자 보존 QA

`extract_numbers()`는 텍스트의 숫자 시퀀스를 등장 순서대로 추출한다. `numbers_preserved()`는 원문과 정제문의 숫자 시퀀스 및 순서가 동일한지 비교한다.

예를 들어 `월 30만원, 3개월`과 `월 30만 원, 3개월`은 동일한 숫자 시퀀스로 판정하지만, `월 300만원, 3개월`로 변경되면 검토 대상으로 판정한다.

### 5-4. 부분 재실행 대상 판별

`needs_llm_rerun()`은 기존 체크포인트의 `주요내용_정제`에 다음 흔적이 남아 있는지 판별한다.

- PUA 문자
- `(지원대상)`, `(지원내용)`, `(사업내용)` 등 괄호형 라벨

해당 함수는 재실행 여부만 반환하며 체크포인트 파일을 직접 수정하지 않는다. 체크포인트 읽기·제거·저장은 연도와 파일 경로를 관리하는 노트북에서 수행한다.

## 6. `text_match.py` 구현 내용

`normalize_for_match()`는 유사도 비교를 위해 공백과 일부 문장부호를 제거한다. `find_near_duplicates()`는 같은 지역·중분류 안에서 완전 일치 및 공백·문장부호만 다른 이름을 제외하고 RapidFuzz 유사도가 임계값 이상인 사업명 쌍을 검토 후보로 반환한다. 후보가 0건이어도 정해진 반환 컬럼을 유지한다.

`dedup_label()`은 괄호형 라벨, 연속 중복 라벨, 라벨 앞 불릿을 정리한다. `check_pattern()`과 `check_pattern_broad()`는 엄격·확장 라벨 형식의 일치 여부를 구분하며, `extract_via_regex()`는 확장 패턴이 확인된 문장에서만 지원대상과 지원내용을 분리한다. 유사 사업명 결과는 자동 병합 판정이 아니라 검토 후보로만 사용한다.

- 결측 입력의 API 호출 방지
- 1회 재시도 후 성공
- 반복 실패 후 원문 유지
- 잘못된 재시도 횟수 차단
- 숫자 시퀀스 추출 및 보존 여부
- PUA·괄호형 라벨 재실행 판별

`text_match.py`는 `tests/test_text_match.py`에서 이름 정규화, 지역·중분류 범위 제한, 공백만 다른 이름 제외, 빈 후보 결과, 라벨 정리, 엄격·확장 패턴 판별과 정규식 추출을 검증했다.

### 7-1. 전용 노트북 통합 검증

기존 연도별 노트북을 수정하지 않고 `notebooks/archive/20260721_공통_유틸_모듈_동작검증.ipynb`를 새로 생성해 세 모듈을 한 커널에서 함께 검증했다. 새 커널에서 전체 셀을 위에서 아래로 실행했고 결과를 노트북에 저장했다.

```text
pipeline_common 검증 통과
text_match 검증 통과
모의 LLM 결과: 월 30만 원씩 3개월 지원
llm_refine 검증 통과
공통 유틸 모듈 노트북 검증 완료
```

검증 과정에서 현재 pandas의 `groupby(...).apply(assign_labels)` 결과에 그룹 컬럼 `지역`이 남지 않는 것을 확인했다. 기존 연도별 노트북과 동일하게 원본 인덱스를 이용해 다음과 같이 지역을 복원해야 한다.

```python
df_labeled = df_raw.groupby("지역", group_keys=False).apply(assign_labels)
df_labeled["지역"] = df_raw.loc[df_labeled.index, "지역"]
```

LLM 검증은 모의 클라이언트로 수행했으며 실제 API 호출과 비용은 발생하지 않았다.

소계 QA는 합성 데이터의 원본 소계 1,200과 leaf 합계 1,000을 비교해 `차이=-200`, `결과=불일치`로 판정하는 것을 확인했다.

실제 연도별 작업에서는 `notebooks/templates/시행계획_시도분리정제_공통유틸_템플릿.ipynb`를 복사해 연도, 입력 파일, 0원 토큰, 반복 머리글 패턴, QA 허용오차를 수정해 사용한다. 실행 결과를 보존한 검증 노트북은 `archive/`, 재사용할 원본은 `templates/`로 분리했다.

## 8. 작업 중 수정한 오류

- `assign_labels()`의 `Series.ea()` 오타를 `Series.eq()`로 수정했다.
- `__all__`이 함수 내부에 포함되지 않도록 모듈 수준에 배치했다.
- 파일 전체 들여쓰기를 Ruff 형식에 맞게 정리했다.
- 파일 끝 개행 누락을 수정했다.
